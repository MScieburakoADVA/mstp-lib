StpCallback_DebugStrOut

void StpCallback_DebugStrOut
(
    const STP_BRIDGE* bridge,
    int               portIndex,
    int               treeIndex,
    const char*       nullTerminatedString,
    unsigned int      stringLength,
    bool              flush
);

Summary

Application-defined callback that processes debug text generated by the STP library. It usually sends this text to a PC for displaying in a program such as Hyperterminal.

StpCallback_DebugStrOut is a placeholder name used throughout this documentation. The application may name this callback differently.

Parameters

bridge
The application receives in this parameter a pointer to the bridge object returned by STP_CreateBridge.
portIndex
The application receives in this parameter either:
  • the zero-based index of a port, if the debug text is relevant only to that port, or
  • -1 if the debug text is relevant to all ports (e.g., contains information about the bridge).
treeIndex
The application receives in this parameter either:
  • the zero-based index of a spanning tree, if the debug text is relevant only to that spanning tree, or
  • -1 if the debug text is relevant to all spanning trees.
nullTerminatedString
The application receives in this parameter a pointer to the null-terminated debug text.
stringLength
The length of the text (the value that strlen() would return).
flush
The application receives in this parameter a value which can be used as a hint for flushing any debug text that the application might have buffered.

Remarks

The length of the debug text passed to this function, including the null terminator, is lower than or equal to the debugLogBufferSize parameter specified in the call to STP_CreateBridge.

Logging is enabled or disabled by calling STP_EnableLogging. When a bridge is created, logging for that bridge is disabled.

Line ending used by the STP library is "\r\n".

The application may choose to buffer the text it receives in this callback. If it does so, it should flush its buffer when the library passes true in the flush parameter.

The library sometimes calls this callback only for flushing, without any text (so nullTerminatedString pointing to a null character, stringLength = 0, and flush = true).
