StpCallback_AllocAndZeroMemory

void* StpCallback_AllocAndZeroMemory (unsigned int size);

Summary

Application-defined function that allocates and zeroes-out memory for STP use. (The library does not call standard memory-allocation functions, because most embedded applications don't have a heap to allocate from.)

StpCallback_AllocAndZeroMemory is a placeholder name used throughout this documentation. The application may name this callback differently.

Parameters

size
The application receives in this parameter the size of the required memory block.

Return value

The application must return a pointer to the allocated and zeroed-out memory block.

Remarks

The STP library does not currenly support out-of-memory conditions. It verifies with an assert() that the pointer returned by this callback is not NULL.

See the remarks section in STP_CreateBridge, for information about how and when the STP library allocates memory.
