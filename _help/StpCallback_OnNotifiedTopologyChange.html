StpCallback_OnNotifiedTopologyChange

void StpCallback_OnNotifiedTopologyChange
(
    const STP_BRIDGE* bridge,
    unsigned int portIndex,
    unsigned int treeIndex,
    unsigned int timestamp,
);

Summary

Application-defined function that is called by the STP library when it detects a change in the network topology somewhere within the network, on a distant bridge that's reachable from a specific local port.

This callback should flush the FDB entries (MAC-to-port associations) on the specified port and specified spanning tree. See also the related StpCallback_FlushFdb.

Parameters

bridge
The application receives in this parameter a pointer to an STP_BRIDGE object.
portIndex
The application receives in this parameter the index of the physical port that in the spanning tree topology was, or is, connected to the point in network where the topology change ocurred.
treeIndex
The application receives in this parameter the zero-based index of the spanning tree on which the topology change ocurred. For STP or RSTP, this is always zero. For MSTP, this is zero for CIST, or 1..64 for an MSTI.
timestamp
The application receives in this parameter the timestamp that it passed to the function that triggerred the detection (usually STP_OnBpduReceived). Useful for debugging and troubleshooting.

Remarks

StpCallback_OnNotifiedTopologyChange is a placeholder name used throughout this documentation. The application may name this callback differently.
