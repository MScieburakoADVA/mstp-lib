STP_SetMstConfigTableAndComputeDigest1

void STP_SetMstConfigTableAndComputeDigest1
(
    STP_BRIDGE*          bridge,
    const unsigned char  mstids [4094],
    unsigned int         timestamp
);

Summary

Sets the MST Config Table (VLAN to MSTID mapping) and computes the Digest field of the MST Configuration Identifier.

Parameters

bridge
Pointer to a STP_BRIDGE object.
mstids
Pointer to an array of 4094 values, where each values specifies the MSTID mapped to a VLAN number.
timestamp
A timestamp used for the debug log.

Remarks

This function may be called only for MSTP bridges (i.e., if STP_VERSION_MSTP was passed to STP_CreateBridge or STP_SetStpVersion).

See §13.7 in 802.1Q-2011 for more information about the MST Configuration Identifier.

The bridge supports only the number of trees that was specified in the call to STP_CreateBridge. The caller must not attempt to map a VLAN to a non-existent tree, i.e., all values in the mstids array must be lower than the treeCount parameter that was specified in the call to STP_CreateBridge. The STP library will raise an assertion if the caller attempts to do that.

The value at index 0 of the mstids array corresponds to VLAN 1, the value at index 1 to VLAN 2, and so on until the value at index 4093, which corresponds to VLAN 4094.

The STP library computes the Digest as described in 13.7.4) in 802.1Q-2011.

If this function is not called, a default mapping is used which is generated by STP_CreateBridge. See the Remarks section there for more information.

This function may not be called from within an STP callback.
