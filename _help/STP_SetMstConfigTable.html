STP_SetMstConfigTable

void STP_SetMstConfigTable
(
    STP_BRIDGE*                   bridge,
    const STP_CONFIG_TABLE_ENTRY* entries,
    unsigned int                  entryCount,
    unsigned int                  timestamp
);

Summary

Sets the MST Config Table (VLAN to MSTID mapping) and computes the Digest field of the MST Configuration Identifier.

Parameters

bridge
Pointer to a STP_BRIDGE object.
table
Pointer to an array of STP_CONFIG_TABLE_ENTRY, where each entry specifies a mapping between a VLAN and its corresponding MSTID. First entry is unused and must be zero; second entry specifies the tree mapped to VLAN 1; third entry specifies the tree mapped to VLAN 2; and so on.
entryCount
The number of entries in table. This must be equal to 1 + maxVlanNumber, where maxVlanNumber is the value that was passes to STP_CreateBridge.
timestamp
A timestamp used for the debug log.

Remarks

This function can be called also when the bridge isn't running MSTP.

See §13.7 in 802.1Q-2011 for more information about the MST Configuration Identifier.

The caller must not attempt to map a VLAN to a non-existent tree, i.e., the table passed to this function must contain values in the treeIndex field that are <=  mstiCount parameter that was specified in the call to STP_CreateBridge. The STP library raises assertions for invalid values.

The STP library computes the Digest as described in 13.7.4) in 802.1Q-2011.

If this function is not called, a default mapping is used which is generated by STP_CreateBridge. See the Remarks section there for more information.

This function may not be called from within an STP callback.

See also STP_GetMstConfigTable.
