STP_SetPortAdminEdge

void STP_SetPortAdminEdge
(
    STP_BRIDGE*   bridge,
    unsigned int  portIndex,
    bool          adminEdge
);

Summary

Sets the AdminEdge parameter for a given port.

Parameters

bridge
Pointer to a STP_BRIDGE object, obtained from STP_CreateBridge.
portIndex
The index of the port for which the AdminEdge parameter is to be set.
newAdminEdge
The new value of the AdminEdge parameter.

Remarks

See §13.25.1 in 802.1Q-2011 for more information.

The current library code was not tested in scenarios where this function is called after the bridge has been started. It was only tested in scenarios where this function is called once, after STP_CreateBridge is called, and before STP_StartBridge is called.

Execution of this function is a potentially lengthy process. It may call various callbacks multiple times.

When a port is configured as Edge by calling this function with adminEdge true, the library will still be sending BPDUs to that port, and will still need to receive and process BPDUs from that port. This is not a characteristic of this particular library; it is a characteristic of the STP protocol itself. This behavior is useful in case the end user configures a port as Edge by mistake (e.g., correctly configures a port as Edge in one location but later moves the switch to another location and doesn't remember to reconfigure that port.)

This function may not be called from within an STP callback.
