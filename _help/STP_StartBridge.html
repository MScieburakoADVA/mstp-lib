STP_StartBridge

void STP_StartBridge
(
    STP_BRIDGE*   bridge,
    unsigned int  timestamp
);

Summary

Starts the Spanning Tree Protocol on a bridge.

Parameters

bridge
Pointer to a STP_BRIDGE object, obtained from STP_CreateBridge.
timestamp
A timestamp used for the debug log.

Remarks

Starting the Spanning Tree Protocol from application code is usually a three-step procedure:

  • a call to STP_CreateBridge, which allocates internal structures and initializes all parameters to default values;
  • various calls to functions such as STP_SetBridgePriority and STP_SetStpVersion, possibly with values read from a configuration file; the protocol is not yet started, so there won't be any STP activity with temporary parameters.
  • a call to this function, which starts the protocol directly with application-specified parameters.

The application must not call this function for already-started bridges.

Execution of this function is a potentially lengthy process. It may call various callbacks multiple times.

This function may not be called from within an STP callback.
